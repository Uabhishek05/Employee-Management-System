Employee Management System (EMS)

Project Documentation | Generated on 07 March 2026

Purpose: This document explains the EMS project features with step-by-step workflows, organized separately for:

  1. Login Page
  2. Admin Dashboard
  3. Employee Dashboard

Tech Stack: React + Vite + Tailwind CSS + LocalStorage persistence.

Project Overview

  • Role-based entry: after login, users are routed to admin or employee dashboard.
  • Data is persisted in browser localStorage using keys like ems_users, ems_tasks, and ems_employees.
  • Task statuses used across the system: new, accepted, completed, failed.
  • Global notifications are stored and shown from a shared notification center.

1. Login Page (Separate Module)

The login screen authenticates users and routes them based on role.

Main Features

  • Email/password authentication using credentials from ems_users.
  • Client-side form validation (required fields, valid email format, minimum password length).
  • Show/Hide password toggle.
  • Remember Me checkbox (UI only in current implementation).
  • Quick-fill credential buttons for all demo accounts.
  • Error feedback for invalid authentication.

Step-by-Step User Flow

  1. User opens EMS and sees the login page.
  2. User enters email and password (or clicks a demo credential button).
  3. User clicks Login.
  4. System validates inputs (empty fields, invalid email, short password).
  5. System checks user in local storage credential list.
  6. If invalid: error message is shown.
  7. If valid: user profile is saved in ems_current_user.
  8. User is redirected automatically:
    • Admin role -> Admin Dashboard
    • Employee role -> Employee Dashboard
Implementation reference: src/components/Auth/Login.jsx, src/context/AuthContext.jsx, src/App.jsx.

2. Admin Dashboard (Separate Module)

Admin dashboard provides full team, task, assignment, analytics, and report controls.

Admin Navigation Areas

  • Dashboard
  • Employees
  • Tasks
  • Analytics
  • Reports
  • Settings

Step-by-Step Feature Workflows

A) Create a New Task

  1. Open Quick Actions -> Create Task.
  2. Enter title, description, priority, due date, and assignee.
  3. Click Submit Task.
  4. System adds task with status new and updates assigned count for the selected employee.
  5. A success notification is added in notification center.

B) Assign/Reassign Task

  1. Open Quick Actions -> Assign Task.
  2. Select task and employee.
  3. Click Assign Task.
  4. Task status becomes accepted and assignment is updated.
  5. Notification is generated for tracking.

C) Delete Task

  1. Open Quick Actions -> Create Task panel.
  2. Switch tab to Delete Task.
  3. Select task and confirm delete.
  4. Task is removed and assigned count is reduced for the original owner.

D) Add Employee

  1. Open Quick Actions -> Add Employee.
  2. Keep action on Add Employee tab.
  3. Enter name, email, and role.
  4. Submit form to add employee with default status Active.

E) Remove Employee

  1. Open Quick Actions -> Add Employee.
  2. Switch to Remove Employee.
  3. Select employee and submit removal.
  4. System reassigns removed employee tasks to a fallback employee.

F) Toggle Employee Status

  1. Go to Employee table.
  2. Click Activate/Deactivate for a user.
  3. Status toggles between Active and On Leave.

G) Generate Reports

  1. Open Quick Actions -> Generate Report.
  2. Select report type (Daily, Weekly, Task Performance).
  3. Download as CSV or PDF.
  4. System creates a downloadable file containing report summary lines.

H) Monitor Dashboard & Analytics

  1. View overview cards: total employees, active tasks, completed, pending, overdue.
  2. Use task board and search to inspect filtered tasks.
  3. Open analytics for task distribution, weekly productivity, and performance.
  4. Track recent activity and upcoming deadlines panels.
Implementation reference: src/components/Dashboard/AdminDashboard.jsx + shared components in src/components/other.

3. Employee Dashboard (Separate Module)

Employee dashboard is focused on personal task execution, status updates, and self-service settings.

Employee Navigation Areas

  • Dashboard
  • Tasks
  • Team
  • Calendar
  • Reports
  • Settings

Step-by-Step Feature Workflows

A) View Personal Task Summary

  1. Login with employee account.
  2. Open Dashboard to see task summary cards.
  3. Click any card (New/Accepted/Completed/Failed) to filter tasks instantly.
  4. Use Show All to clear filters.

B) Update Task Status

  1. Go to Tasks tab.
  2. Find assigned task card.
  3. Click one of actions: Accept, Complete, or Fail.
  4. Task status is updated in global task store.
  5. Notification entry is created with user and task status change.

C) Search and Filter Tasks

  1. Use top search input to filter by task title, description, or priority.
  2. Combine search with status filter cards for precise results.

D) Review Team and Deadlines

  1. Open Team tab to see coworker status and performance counts.
  2. Open Calendar tab to track upcoming task deadlines.

E) View Reports and Analytics

  1. Open Reports tab.
  2. View task distribution chart and weekly productivity bars for current user tasks.

F) Manage Personal Settings

  1. Open Settings.
  2. Check profile details (name, email, current status).
  3. Click Take Leave / Return to Work to toggle status.
  4. Click Logout when done.

G) Notifications and Profile Menu

  1. Use bell icon in header to read notifications.
  2. Use profile menu for View Profile and Logout.
  3. Use Clear inside notification dropdown to remove all notifications.
Implementation reference: src/components/Dashboard/EmployeeDashboard.jsx + shared Header, Sidebar, TaskList, TaskListNumbers, AnalyticsSection.

Data & Persistence Notes

  • All app data is client-side in browser storage.
  • On first run, seed data is initialized automatically from defaults.
  • No backend/API is required for current project execution.

Run Instructions

  1. Open project folder EMS.
  2. Install dependencies: npm install
  3. Start development server: npm run dev
  4. Open the local Vite URL in browser.
